# Suggested docs structure

Use this as a target structure if the docs are reorganized in the future. Keep the top-level
`docs/README.md` as the map to every section.

```
docs/
├── README.md
├── agents/
│   ├── README.md
│   ├── workflow-and-checks.md
│   ├── formatting-and-tooling.md
│   ├── repo-orientation.md
│   ├── coding-practices.md
│   ├── version-control.md
│   └── review-checklist.md
├── architecture/
│   ├── overview.md
│   ├── deployment.md
│   └── middleware.md
├── content/
│   ├── adding-pages.md
│   ├── content-components.md
│   └── content-data.md
├── contributor/
│   ├── contributor-workflow.md
│   └── local-development.md
├── diagnostics/
│   ├── diagnostics-capacity-forecaster.md
│   └── diagnostics-outputs.md
├── design/
│   ├── visual-textures.md
│   └── design-references.md
├── testing/
│   ├── testing-todos.md
│   └── cloudflare-playwright.md
└── ux/
    └── usability-audit.md
```
